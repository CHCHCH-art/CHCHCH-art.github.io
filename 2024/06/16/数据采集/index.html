kafka单节点配置（单节点单broker

server.properties文件中，需要修改两处配置：listeners 和 log.dirs

listeners：服务器监听的地址，修改如下：

listeners=PLAINTEXT://localhost:9092

log.dirs：日志文件修改为自定义的日志目录，我的是log.dirs=D:/kafka_2.12-3.1.0/logs

zookeeper配置
zookeeper配置文件为zookeeper.properties，只需修改一处：

dataDir：zookeeper存储数据的路径，Windows环境路径要用D:\kafka3.2.1\datas这种形式

添加：audit.enable=true

为了保证kafka不会出现命令行问题，首先考虑把kafka放到C盘下面，然后修改文件夹名字为kafka不要附带任何版本信息

然后新建一个data、logs文件夹，后面需要设置目录，然后上面的斜杠自己看清楚，电脑上并不会自动保存，需要自己去进行保存

zooker服务器先：

.\bin\windows\zookeeper-server-start.bat .\config\zookeeper.properties

kafka服务器后：

.\bin\windows\kafka-server-start.bat .\config\server.propertie

主题创建：（具体根据需要修改命令

start cmd /k .\bin\windows\kafka-topics.bat –create –bootstrap-server 10.0.102.132:9092 –replication-factor 1 –partitions 1 –topic test

启动一个生产者：

.\bin\windows\kafka-console-producer.bat –broker-list localhost:9092 –topic test

启动一个消费者：

.\bin\windows\kafka-console-consumer.bat –bootstrap-server localhost:9092 –topic test

具体消费者那边可能不能一次性出来结果，在cmd界面打一个回车就出来了

参考文献：https://blog.csdn.net/qq_42887496/article/details/126244049

多节点配置：（多节点多broker）
server-1.properties文件
#broker的全局唯一编号，不能重复
broker.id=1
#监听
listeners=PLAINTEXT://:9093
#日志目录
log.dirs=/home/hadoop/kafka-logs-1

复制三个log文件夹分别命名为log1/log2/log3，复制三个propertiers文件，分别命名为server1.properties、server2.properties、server3.properties

对应修改各个server.properties里面log的内容，broker的id、各个监听地址为9092、9093、9094

zookeeper服务启动：

.\bin\windows\zookeeper-server-start.bat .\config\zookeeper.properties

三个kafka分别需要cmd启动：

.\bin\windows\kafka-server-start.bat .\config\server1.properties

.\bin\windows\kafka-server-start.bat .\config\server2.properties

.\bin\windows\kafka-server-start.bat .\config\server3.properties

启动一个生产者：

.\bin\windows\kafka-console-producer.bat –broker-list localhost:9092 –topic test

启动两个消费者：

.\bin\windows\kafka-console-consumer.bat –bootstrap-server localhost:9093 –topic test

.\bin\windows\kafka-console-consumer.bat –bootstrap-server localhost:9094 –topic test

具体也可以创建两个生产者一个消费者，反正只要保证都在一个主题下面任意一个消费者都能接受到该主题中所有生产者生产的消息

先从输出里面找到插入\更新，然后对这个进行修改，获取字段是肯定要获取字段的
然后还要获取和更新字段，但是这边可以选择只更新某些字段而另外一些不更新这样的
连接了以后不用和表输出相互连接，否则就会导致再度输出一遍在表内
这个“更新”的含义就是新内容是插入进入的，而不是覆盖原先对应的记录（没法覆盖的）
